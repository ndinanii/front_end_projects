Introduction to Python

Python is an interpreted, high-level, general-purpose programming language. Created by Guido van Rossum and first released in 1991, Python's design philosophy emphasizes code readability with its notable use of significant indentation.

It is dynamically typed and garbage-collected, supporting multiple programming paradigms, including structured (procedural), object-oriented, and functional programming. Python is widely used for web development, data science, artificial intelligence, and more.

print("Welcome to Python!")
Basic Syntax

Python's syntax is designed to be clear and concise. Key elements include:

  • Indentation: Python uses indentation (typically four spaces) to define blocks of code, rather than curly braces.
  • Comments: Single-line comments start with a #. Multi-line strings (using triple quotes ''' or """) are often used for multi-line comments.
  • Case-Sensitivity: Python is case-sensitive, meaning variable and Variable are distinct.

Variables are created when you assign a value to them, and you do not need to declare their type explicitly.

# This is a comment name = "Python" # A variable holding a string year = 1991 # A variable holding an integer if year > 1990: print(f"{name} was released after 1990.")
Variables and Data Types

Variables are names that refer to values in memory. Python has several built-in data types:

  • Numeric Types: int (integers), float (floating-point numbers), complex (complex numbers).
  • Text Type: str (strings).
  • Sequence Types: list (ordered, mutable), tuple (ordered, immutable), range.
  • Mapping Type: dict (unordered key-value pairs).
  • Set Types: set (unordered, unique items), frozenset (immutable set).
  • Boolean Type: bool (True or False).

The type of a variable is determined at runtime (dynamic typing).

age = 30 # int price = 19.99 # float name = "Alice" # str colors = ["red", "green"] # list person = {"name": "Bob", "age": 25} # dict is_valid = True # bool
Control Flow

Control flow statements manage the order in which the program's code executes.

Conditional statements like if, elif (else if), and else allow you to execute code blocks based on conditions. Loops like for (to iterate over sequences) and while (to repeat as long as a condition is true) are used for repetitive tasks.

x = 10 if x > 5: print("x is greater than 5") elif x == 5: print("x is 5") else: print("x is less than 5") # For loop for i in range(3): # Iterates 0, 1, 2 print(f"Loop iteration: {i}") # While loop count = 0 while count < 2: print(f"While count: {count}") count += 1

Loop control statements like break (exit loop) and continue (skip to next iteration) can modify loop behavior.

Functions

Functions are reusable blocks of code that perform a specific task. They are defined using the def keyword.

Functions can take input parameters (arguments) and can return output values using the return statement. This helps in organizing code, making it modular and easier to maintain.

# Defining a function def greet(name): return f"Hello, {name}!" # Calling the function message = greet("Developer") print(message) # Output: Hello, Developer! def add(a, b=1): # b has a default value return a + b sum_result = add(5) print(sum_result) # Output: 6 sum_result_two = add(5, 3) print(sum_result_two) # Output: 8

Docstrings (documentation strings) are triple-quoted strings used as the first statement in a function to describe what it does.

Error Handling

Python uses exceptions to manage errors that occur during program execution. The try...except block is used to catch and handle these exceptions gracefully.

  • The try block contains code that might raise an exception.
  • The except block catches specific exceptions and executes code to handle them.
  • The else block (optional) executes if no exceptions occur in the try block.
  • The finally block (optional) always executes, regardless of whether an exception occurred (useful for cleanup).

This prevents the program from crashing and allows for more robust applications.

try: numerator = 10 denominator = 0 # This will cause a ZeroDivisionError # denominator = 2 # Try this for a successful division result = numerator / denominator except ZeroDivisionError: print("Error: Cannot divide by zero!") result = None except TypeError: print("Error: Invalid data type for operation.") result = None else: print("Division was successful.") finally: print("Try-except block finished.") if result is not None: print(f"Result: {result}")
